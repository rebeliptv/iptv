## What's New

- **Security:** updated the bundled Docker components used by the in-app updater to the latest release, clearing several published advisories. No change to how the app works.
- **Clicking a channel in the TV Guide or on the Dashboard now opens its channel page** (with details and a player) instead of popping a quick preview.
- **Custom-source channels now number from 10,000 and count up contiguously.** Channels from your added sources are numbered as one tidy run starting at 10,000 — each new source's channels follow on from the last — instead of each source jumping by a million.
- **Optionally show channel numbers in your playlist.** A new **Settings → Player** toggle advertises each channel's number to Jellyfin/Plex, so your lineup sorts by number instead of alphabetically by name. Off by default (your current lineup is unchanged until you turn it on).
- **More resilient local database.** Brief internal "database is locked" hiccups are now retried instead of surfacing as errors, and if the database ever becomes corrupted the app rebuilds it automatically on the next start instead of getting stuck.
- **Live channels no longer freeze repeatedly mid-watch in Jellyfin.** Some channels would freeze — picture and sound together — for a few seconds and recover, again and again. The app now keeps a short playback buffer instead of running right at the live edge, so a momentary delivery hiccup no longer drains the picture.
- **Live channels ride out a flaky provider instead of dropping you.** A brief hiccup (a slow segment or an ad break) no longer ends the channel and forces a reconnect — the stream waits it out — and if a source genuinely goes down, the channel switches to a backup instead of retrying the dead one.
- **The dashboard's built-in player recovers from more playback glitches.** A momentary decode or buffer hiccup used to stop playback outright (you'd have to click play again); it now reopens automatically from a fresh edge, giving up only if the feed genuinely can't be played.

## Docker

```bash
docker pull rebeliptv/iptv:1.3.1
```

### Docker Compose

```yaml
services:
  iptv:
    image: rebeliptv/iptv:1.3.1
    hostname: iptv
    container_name: iptv
    ports:
      - "8080:8080"
    # Environment variables are OPTIONAL — defaults work out of the box.
    # Uncomment any you need:
    # environment:
    #   TZ: "America/New_York"   # set to YOUR timezone, e.g. Europe/London, Etc/UTC
    volumes:
      - iptv-data:/app/data
    dns:
      - 8.8.8.8
      - 4.2.2.2
      - 1.1.1.1
    stop_grace_period: 30s
    restart: unless-stopped

volumes:
  iptv-data:
```

### Docker Run

```bash
docker run -d \
  --name iptv \
  -p 8080:8080 \
  -v iptv-data:/app/data \
  --dns 8.8.8.8 \
  --dns 4.2.2.2 \
  --dns 1.1.1.1 \
  --restart unless-stopped \
  rebeliptv/iptv:1.3.1

# Environment variables are optional (defaults work). Add -e flags as needed, e.g.:
#   -e TZ=America/New_York   (set to YOUR timezone — e.g. Europe/London, Etc/UTC)
```

### Updating

```bash
docker pull rebeliptv/iptv:1.3.1
docker compose up -d
```

#### In-app updates (optional)

To upgrade from the dashboard's **Update now** button instead of the
command line, run the published `:latest` image and mount the Docker
socket so the container can update itself:

```yaml
services:
  iptv:
    image: rebeliptv/iptv:latest
    hostname: iptv
    container_name: iptv
    ports:
      - "8080:8080"
    # Environment variables are OPTIONAL — defaults work out of the box.
    # Uncomment any you need:
    # environment:
    #   TZ: "America/New_York"   # set to YOUR timezone, e.g. Europe/London, Etc/UTC
    volumes:
      - iptv-data:/app/data
      - /var/run/docker.sock:/var/run/docker.sock
    dns:
      - 8.8.8.8
      - 4.2.2.2
      - 1.1.1.1
    stop_grace_period: 30s
    restart: unless-stopped

volumes:
  iptv-data:
```

> Mounting the Docker socket gives the container control of the Docker
> daemon on the host — enable it only if you're comfortable with that.
> The button appears only on the `:latest` image when a newer release
> exists; it's hidden on version-pinned images.

## Endpoints

| URL | Description |
|-----|-------------|
| `http://<server>:8080/playlist?key=YOUR_KEY` | M3U playlist |
| `http://<server>:8080/epg?key=YOUR_KEY` | EPG guide (XML) |
| `http://<server>:8080/epg.gz?key=YOUR_KEY` | EPG guide (gzip) |
| `http://<server>:8080` | Web dashboard |

> **Note:** The `?key=` parameter is only required if an API key has been generated in the web dashboard under **Settings**.
