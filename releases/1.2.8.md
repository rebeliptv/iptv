## What's New

- **New: choose where your channels and guide come from.** A Settings toggle lets you scrape providers locally (the default) or stream everything — channels, guide and live sports — through the Rebel IPTV relay. If the usual sources are blocked, throttled or unreliable in your region, switching to the relay keeps things working; flip it back any time and your guide rebuilds automatically.
- Broader coverage out of the box — added new channel and program-guide sources spanning more US national networks, local stations, sports, premium/movie and kids channels, so your lineup and listings keep populating even if one source goes quiet.
- The Program Guide now shows only channels you can actually watch — no more empty rows for offline channels or ones with no program information.
- The Program Guide loads much faster, and an unchanged guide is no longer re-downloaded.
- Program Guide display fixes — movies are highlighted in purple more consistently, and short programs no longer render too wide and overlap the next one.
- Live-stream playback fixes — audio and video re-align after a stream buffers and resumes, and a feed stuck showing a provider "no signal" card (picture but no sound) now switches to a backup source automatically.
- Custom M3U sources now refresh in parallel, so one slow or unreachable source no longer stalls the rest.
- Fixed live sports events occasionally showing "no feed" even while live — they now pair with the correct stream whatever timezone you're in.
- Automatically uses HTTP/3 (QUIC) with upstreams that support it for a possible speed-up; everything else is unchanged.
- Uses less memory, so it runs comfortably on lower-RAM systems.

## Docker

```bash
docker pull rebeliptv/iptv:1.2.8
```

### Docker Compose

```yaml
services:
  iptv:
    image: rebeliptv/iptv:1.2.8
    hostname: iptv
    container_name: iptv
    ports:
      - "8080:8080"
    # Environment variables are OPTIONAL — defaults work out of the box.
    # Uncomment any you need:
    # environment:
    #   TZ: "America/New_York"   # set to YOUR timezone, e.g. Europe/London, Etc/UTC
    #   USE_PROXY: "true"        # only if providers are blocked in your region
    volumes:
      - iptv-data:/app/data
    dns:
      - 8.8.8.8
      - 4.2.2.2
      - 1.1.1.1
    stop_grace_period: 30s
    restart: unless-stopped

volumes:
  iptv-data:
```

### Docker Run

```bash
docker run -d \
  --name iptv \
  -p 8080:8080 \
  -v iptv-data:/app/data \
  --dns 8.8.8.8 \
  --dns 4.2.2.2 \
  --dns 1.1.1.1 \
  --restart unless-stopped \
  rebeliptv/iptv:1.2.8

# Environment variables are optional (defaults work). Add -e flags as needed, e.g.:
#   -e TZ=America/New_York   (set to YOUR timezone — e.g. Europe/London, Etc/UTC)
#   -e USE_PROXY=true        (only if providers are blocked in your region)
```

### Updating

```bash
docker pull rebeliptv/iptv:1.2.8
docker compose up -d
```

## Endpoints

| URL | Description |
|-----|-------------|
| `http://<server>:8080/playlist?key=YOUR_KEY` | M3U playlist |
| `http://<server>:8080/epg?key=YOUR_KEY` | EPG guide (XML) |
| `http://<server>:8080/epg.gz?key=YOUR_KEY` | EPG guide (gzip) |
| `http://<server>:8080` | Web dashboard |

> **Note:** The `?key=` parameter is only required if an API key has been generated in the web dashboard under **Settings**.
