## What's New

- **Channels no longer stay "playing" after you stop watching.** Stopping a channel in Jellyfin/Plex — or clicking off a dashboard preview before it finished loading — now promptly releases the feed instead of leaving it running in the background and still counted as an active viewer. A player that keeps its connection open but stops watching is dropped after a short idle window (it reconnects instantly if you resume).
- **The dashboard no longer hangs on "Initializing".** That loading screen (which waited on a health check and could get stuck there — most often after signing in behind a reverse proxy) is gone. The dashboard now loads immediately and each section fills in on its own. Enabling the optional login and reaching the dashboard through a proxy also works now.
- **Stream buffering now smooths out provider hiccups instead of just delaying them.** With **Settings → Playback → Stream buffering** on, the agent reads the feed a few seconds ahead into memory and hands it to the player at a steady rate — so the provider's bursty delivery and brief stalls (shorter than your buffer setting) are ironed out before they reach the screen, rather than freezing the picture. Costs a few seconds of start-up delay and some server memory per active viewer (scaled to the seconds you pick). Note: this can't hide a provider drop that *ends* the stream (the player reconnects) — only genuine slowdowns.
- **Live sports stutter and freeze far less.** Game feeds arrive in choppier bursts than regular channels, so the app now automatically keeps a little more buffer for events — smoothing out the bursts that used to freeze the picture during a game. It's automatic (nothing to turn on), and regular channels still start quickly.
- **Smoother when several people watch the same channel.** The app now pulls a channel from the provider once and shares that one feed with everyone watching it, instead of opening a separate connection for each viewer. Less load on the provider means fewer hiccups and less buffering — and a viewer who joins a channel someone's already watching starts almost instantly.

## Update

Already running Rebel IPTV? Update to this version — your data (the `iptv-data` volume) is kept. From the dashboard, click **Update now** on the Dashboard or in **Settings → Server** (on the `:latest` image with the Docker socket mounted). Or from the command line:

```bash
docker pull rebeliptv/iptv:1.3.3
docker compose up -d
```

Setting up a **new** instance? The step-by-step guide at **https://www.rebeliptv.net** builds a Docker configuration for your system — ports, timezone, in-app updates, and more — so you copy a setup that fits your box instead of a generic one.

Image: `rebeliptv/iptv:1.3.3` (also published as `rebeliptv/iptv:latest`).

## Endpoints

| URL | Description |
|-----|-------------|
| `http://<server>:8080/playlist?key=YOUR_KEY` | M3U playlist |
| `http://<server>:8080/epg?key=YOUR_KEY` | EPG guide (XML) |
| `http://<server>:8080/epg.gz?key=YOUR_KEY` | EPG guide (gzip) |
| `http://<server>:8080` | Web dashboard |

> **Note:** The `?key=` parameter is only required if an API key has been generated in the web dashboard under **Settings**.
