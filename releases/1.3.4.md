## What's New

- **The live "active streams" count now reflects how many people are actually watching.** Two viewers on the same channel now count as two, not one — previously multiple viewers of one channel showed as a single active stream.
- **Live sports events now correctly show as watchable only when their feed is actually live.** Previously an event could show a "Watch Now" the moment its scheduled start passed, even if the feed wasn't up yet or the game had already finished; each is now health-checked against the real stream, so a dead tile no longer appears.
- **More live sports-event sources.** More sources now back live games, so more games have a working feed and there's more automatic failover when one goes down — and many now play directly in the app, with no extra software needed.
- **Playback no longer freezes when a feed reconnects.** Providers hand out a fresh feed every minute or two, and that hand-over used to corrupt the picture — in Jellyfin it dropped to a heavy re-encode it couldn't keep up with and froze for good; in the dashboard it froze and resumed, broke up every ~80 seconds, or went to a black screen. Feeds now join cleanly, so the picture continues straight through the switch. **Jellyfin users: turn OFF the tuner's "Auto-loop live streams" setting** — the reconnect is now seamless, and leaving it on is what forces that freezing re-encode (see the README's Jellyfin setup).
- **Streams recover on their own.** A live feed routinely gets dropped and reissued by the provider (often about once an hour); the agent now re-opens it behind the scenes and carries your picture straight across the gap, so those hand-offs no longer freeze or interrupt playback. And when a source genuinely goes bad, the stream reconnects onto a working backup instead of the dead one — clearing up streams that could hang indefinitely or a slow source quietly running out of video.
- **Every channel and event plays as one smooth, continuously-buffered stream.** Feeds arrive in several different technical formats; the agent now delivers all of them to your player as a single continuous stream and buffers each one on the server, so playback behaves identically whether you watch in the dashboard or in your own player (Jellyfin, Plex, and similar). An optional buffer under **Settings → Playback → Stream buffering** adds extra cushion for a flaky source.
- **Channels now show which country they're from.** Each channel card in the dashboard carries a small country code (US, CA, UK) in the corner, so you can tell at a glance where a channel comes from.

## Update

Already running Rebel IPTV? Update to this version — your data (the `iptv-data` volume) is kept. From the dashboard, click **Update now** on the Dashboard or in **Settings → Server** (on the `:latest` image with the Docker socket mounted). Or from the command line:

```bash
docker pull rebeliptv/iptv:1.3.4
docker compose up -d
```

Setting up a **new** instance? The step-by-step guide at **https://www.rebeliptv.net** builds a Docker configuration for your system — ports, timezone, in-app updates, and more — so you copy a setup that fits your box instead of a generic one.

Image: `rebeliptv/iptv:1.3.4` (also published as `rebeliptv/iptv:latest`).

## Endpoints

| URL | Description |
|-----|-------------|
| `http://<server>:8080/playlist?key=YOUR_KEY` | M3U playlist |
| `http://<server>:8080/epg?key=YOUR_KEY` | EPG guide (XML) |
| `http://<server>:8080/epg.gz?key=YOUR_KEY` | EPG guide (gzip) |
| `http://<server>:8080` | Web dashboard |

> **Note:** The `?key=` parameter is only required if an API key has been generated in the web dashboard under **Settings**.
