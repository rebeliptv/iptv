## What's New

- More reliable channel loading — intermittent network hiccups that could drop a channel during a refresh are now far less likely
- Guide info now stays current while you're watching — the channel and event detail pages refresh in the background every 5 minutes and the "Now Playing" programme advances on its own, with no page reload
- The "Refreshing" indicator in the top bar now reliably appears whenever the server is refreshing — previously it could stay hidden after a refresh was kicked off
- For Jellyfin: enable the **Auto-loop live streams** tuner setting so playback reconnects automatically after a server restart or update (documented under Usage → Jellyfin)

## Docker

```bash
docker pull rebeliptv/iptv:1.2.2
```

### Docker Compose

```yaml
services:
  iptv:
    image: rebeliptv/iptv:1.2.2
    hostname: iptv
    container_name: iptv
    ports:
      - "8080:8080"
    environment:
      TZ: "America/New_York"
    volumes:
      - iptv-data:/app/data
    dns:
      - 8.8.8.8
      - 4.2.2.2
      - 1.1.1.1
    stop_grace_period: 30s
    restart: unless-stopped

volumes:
  iptv-data:
```

### Docker Run

```bash
docker run -d \
  --name iptv \
  -p 8080:8080 \
  -e TZ=America/New_York \
  -v iptv-data:/app/data \
  --dns 8.8.8.8 \
  --dns 4.2.2.2 \
  --dns 1.1.1.1 \
  --restart unless-stopped \
  rebeliptv/iptv:1.2.2
```

### Updating

```bash
docker pull rebeliptv/iptv:1.2.2
docker compose up -d
```

## Endpoints

| URL | Description |
|-----|-------------|
| `http://<server>:8080/playlist?key=YOUR_KEY` | M3U playlist |
| `http://<server>:8080/epg?key=YOUR_KEY` | EPG guide (XML) |
| `http://<server>:8080/epg.gz?key=YOUR_KEY` | EPG guide (gzip) |
| `http://<server>:8080` | Web dashboard |

> **Note:** The `?key=` parameter is only required if an API key has been generated in the web dashboard under **Settings**.
