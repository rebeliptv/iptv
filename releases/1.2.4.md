## What's New

- Refreshes no longer interrupt what you're watching (especially on Jellyfin) — a refresh's behind-the-scenes health check used to be able to drop an active stream, and a brief window after could block reconnecting. The refresh now leaves any channel or event with a live viewer alone and trusts your stream as proof it's working, so playback keeps going right through a refresh

## Docker

```bash
docker pull rebeliptv/iptv:1.2.4
```

### Docker Compose

```yaml
services:
  iptv:
    image: rebeliptv/iptv:1.2.4
    hostname: iptv
    container_name: iptv
    ports:
      - "8080:8080"
    # Environment variables are OPTIONAL — defaults work out of the box.
    # Uncomment any you need:
    # environment:
    #   TZ: "America/New_York"   # set to YOUR timezone, e.g. Europe/London, Etc/UTC
    #   USE_PROXY: "true"        # only if providers are blocked in your region
    volumes:
      - iptv-data:/app/data
    dns:
      - 8.8.8.8
      - 4.2.2.2
      - 1.1.1.1
    stop_grace_period: 30s
    restart: unless-stopped

volumes:
  iptv-data:
```

### Docker Run

```bash
docker run -d \
  --name iptv \
  -p 8080:8080 \
  -v iptv-data:/app/data \
  --dns 8.8.8.8 \
  --dns 4.2.2.2 \
  --dns 1.1.1.1 \
  --restart unless-stopped \
  rebeliptv/iptv:1.2.4

# Environment variables are optional (defaults work). Add -e flags as needed, e.g.:
#   -e TZ=America/New_York   (set to YOUR timezone — e.g. Europe/London, Etc/UTC)
#   -e USE_PROXY=true        (only if providers are blocked in your region)
```

### Updating

```bash
docker pull rebeliptv/iptv:1.2.4
docker compose up -d
```

## Endpoints

| URL | Description |
|-----|-------------|
| `http://<server>:8080/playlist?key=YOUR_KEY` | M3U playlist |
| `http://<server>:8080/epg?key=YOUR_KEY` | EPG guide (XML) |
| `http://<server>:8080/epg.gz?key=YOUR_KEY` | EPG guide (gzip) |
| `http://<server>:8080` | Web dashboard |

> **Note:** The `?key=` parameter is only required if an API key has been generated in the web dashboard under **Settings**.
