## What's New

- **Buffer your live feeds by a chosen number of seconds.** A new **Settings → Playback → Stream buffering** control lets you trade start-up speed for steadier playback. Leave it **off** to stay as close to live as possible (the default), or switch it on and pick how many seconds to hold in memory (default **15**) so a brief provider hiccup drains that cushion instead of freezing the picture. Works for live channels in both local and relay mode. Note: a bigger buffer also means a channel starts roughly that many seconds later — that delay is the buffer. Takes effect the next time a channel reconnects.
- **The Settings page is reorganized into clearer sections.** Settings are now grouped as **Server**, **Sources**, **Connect**, **Playback**, and **Access** so related options sit together — your feed source (Rebel relay vs. local scraping) now leads the Sources section, and the player-facing options are split into **Connect** (playlist/EPG URLs and how to hook up Jellyfin/Plex) and **Playback** (channel numbers, stream buffering). Each section is also bookmarkable now — the URL remembers which one you're on.

## Update

Already running Rebel IPTV? Update to this version — your data (the `iptv-data` volume) is kept. From the dashboard, click **Update now** on the Dashboard or in **Settings → Server** (on the `:latest` image with the Docker socket mounted). Or from the command line:

```bash
docker pull rebeliptv/iptv:1.3.2
docker compose up -d
```

Setting up a **new** instance? The step-by-step guide at **https://www.rebeliptv.net** builds a Docker configuration for your system — ports, timezone, in-app updates, and more — so you copy a setup that fits your box instead of a generic one.

Image: `rebeliptv/iptv:1.3.2` (also published as `rebeliptv/iptv:latest`).

## Endpoints

| URL | Description |
|-----|-------------|
| `http://<server>:8080/playlist?key=YOUR_KEY` | M3U playlist |
| `http://<server>:8080/epg?key=YOUR_KEY` | EPG guide (XML) |
| `http://<server>:8080/epg.gz?key=YOUR_KEY` | EPG guide (gzip) |
| `http://<server>:8080` | Web dashboard |

> **Note:** The `?key=` parameter is only required if an API key has been generated in the web dashboard under **Settings**.
